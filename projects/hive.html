HIVE

A self-hosted AI "second brain" — a private, retrieval-augmented knowledge assistant that turns an organization's internal documents into a searchable knowledge base its team queries in plain language. Documents, embeddings, and inference never leave the host.

Why it exists

Small studios accumulate institutional knowledge in scattered files — brand guides, client briefs, process docs — and lose hours re-finding it. HIVE was built as the shared memory HoneyBridge's team queries instead of digging through folders. Because the studio handles client material, the hard constraint was privacy: everything runs on local infrastructure, with no data leaving the building.

The assistant answers strictly from the knowledge base, cites its sources, and admits uncertainty instead of guessing — an explicit anti-hallucination posture enforced in the agent's system prompt and retrieval design.

How it works

Ingestion

A Local File Trigger watches a pending folder. Each dropped document is parsed by Docling — OCR plus structural extraction — into markdown with extracted, referenced images (multimodal). Content is split with a markdown-aware recursive splitter, embedded with nomic-embed-text via Ollama, and upserted into the Qdrant hivebrain collection. Extracted images are rewritten to HTTP URLs and served so the assistant can link to them.

The assistant

Questions from the bundled chat UI reach an AI Agent whose system prompt requires it to query hivebrain (top-K retrieval as a tool) before answering. It returns a grounded response with inline image links and source attribution — and says when the knowledge base has nothing relevant rather than inventing an answer. Chat runs on a local Gemma model via Ollama, with an OpenAI model wired in as a drop-in alternative.

  • One-command, hardware-aware deployment. Compose profiles for CPU, NVIDIA, and AMD GPU hosts; workflows import and models pull automatically on first up.
  • Cross-platform setup scripts. Idempotent bootstrap for macOS (Homebrew) and Windows (PowerShell + winget) takes a fresh machine to a running stack.
  • Operational maturity. Documented backup/restore of the two stateful volumes (PostgreSQL + Qdrant), secrets isolated in a gitignored .env, encryption-key discipline spelled out.
  • Extensible by design. A supporting image-processing pipeline runs alongside, and new agents plug into the same stack.

Engineering notes

Component choices weren't defaults — local vs. hosted LLMs, embedding models, chunking parameters, and containerized services were benchmarked against latency, cost, and accuracy tradeoffs before the stack settled.

The 79% tooling-cost reduction comes from replacing per-seat SaaS AI subscriptions with a stack the studio owns; the privacy guarantee is architectural, not contractual.
