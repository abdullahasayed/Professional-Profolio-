Code-Atlas

An intelligent codebase visualizer. Point it at any repository — a Git URL or a local folder — and glide from a bird's-eye architectural map down to individual statement blocks, with cross-linked explanations at every level.

The idea: semantic zoom

Maps don't show every street at country scale — they reveal detail as you approach. Code-Atlas applies the same principle to software. It builds a five-level hierarchy and lets you move through it fluidly: zoom in and containers expand in place to reveal their internals; zoom out and they fold away. This is true semantic zoom — detail is revealed and hidden, not pixel-scaled.

Every explanation cross-references related units — callers, callees, imports, subclasses — and every reference is a link that navigates and re-centers the map. Explanations are generated bottom-up: a module's summary synthesizes its files' summaries, so higher levels are grounded rather than hallucinated.

Built to work offline, upgrade gracefully

Explanations come from a pluggable Explainer with four backends resolved in order: Anthropic, any OpenAI-compatible server, Ollama, and a structural fallback that derives real explanations from the AST and relationship graph — signatures, control flow, callers/callees, hotspots — clearly badged as structural. The app is fully useful with no keys at all.

Every explanation is cached on disk, keyed by sha256(prompt_version | level | code-hash) — unchanged code is never re-explained, which is what makes large repositories affordable. Structural entries are transparently upgraded the moment a live backend is configured.

  • Full AST parsing for 8 languages — Python, JavaScript, TypeScript/TSX, Java, Go, C, C++, Rust — with best-effort regex symbol detection for 15+ more.
  • Edges stored at the finest level (calls, imports, inheritance) and aggregated on demand to whatever nodes are visible, so arrows stay meaningful at every zoom depth.
  • Hand-rolled SVG canvas instead of a graph library — in-place container expansion is the whole point, and no off-the-shelf layout engine does it.
  • Self-demonstrating. Ships with a curated sample repo (200+ units, every explanation cross-linked) and visualizes its own source.
  • Pragmatic ingestion: shallow clones, vendored-directory skip-lists, binary sniffing, size caps — big repos never freeze the UI.

Engineering notes

Notable choices, with rationale kept in the repo's build log: SQLite over JSON for indexes; official per-language tree-sitter wheels over runtime grammar downloads; highlight.js over Monaco. Each trades novelty for reliability where reliability is what the tool sells.

The live LLM path for all three protocols is exercised against a mock server in CI-style scripts — so "optional" doesn't mean "untested."
