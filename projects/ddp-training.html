DDP Training

An end-to-end PyTorch Distributed Data Parallel baseline on Kubernetes: fine-tuning a transformer stance classifier across 2 nodes × 8 NVIDIA B200 GPUs with torchrun — from multi-arch container build to artifact recovery off the cluster.

Why it exists

Single-GPU fine-tuning is a solved tutorial; multi-node training is where real clusters break. This project is the deliberately-scoped baseline you run before anything expensive: a small model (DistilRoBERTa) on a public dataset, so every failure you hit is an infrastructure failure — image architecture, GPU kernel compatibility, pod scheduling, NCCL rendezvous — not a modeling one.

It was built while assessing cluster behavior for SimpliSolve: the same engagement that produced the OCI Cluster Evaluation toolkit. The classifier itself connects back to the stance-analysis thread running through TrackRecord.

The parts that actually bite

The repo's value is in the operational edges most tutorials skip. The image must be built --platform linux/amd64 from an Apple Silicon laptop or pods die with manifest-platform errors. B200s require a PyTorch build with sm_100 kernels — hence CUDA 12.8 + PyTorch nightly, verified with a one-liner before anything touches the cluster. Only global rank 0 writes the model, tokenizer, metrics, and run config to the PVC; a throwaway busybox pod recovers the artifacts afterward.

  • Reproducible runtime. One Dockerfile pins the CUDA/PyTorch combination so every environment — laptop validation, registry, cluster — runs identical code.
  • Clean separation of concerns. Build/push and kubectl apply can happen on different machines; the cluster only needs a manifest and a pullable tag.
  • Debugging runbook included. ImagePullBackOff, platform mismatches, sm_100 kernel errors, pending pods, DDP rendezvous timeouts — each mapped to its cause and fix.
  • A tuning ladder, not a dead end: swap stance topics, scale epochs, upgrade to RoBERTa-base, then remove NCCL_IB_DISABLE=1 and tune InfiniBand/RDMA once the baseline is trusted.

Engineering notes

Baseline-first discipline: NCCL over IB/RDMA is deliberately disabled for the first run, so the initial signal is "does distributed training work at all," decoupled from fabric tuning. Performance comes after correctness.

Even the runbook's smallest entry earns its place — copy-pasted commands failing on hidden non-ASCII spaces is exactly the kind of thing that costs an hour on a cluster billing by the minute.
